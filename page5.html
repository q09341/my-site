Язык SQL, ODBC, JDBC

Язык SQL (Structured Query Language) представляет собой специализированный язык программирования, используемый для управления и работы с реляционными базами данных. Он позволяет создавать, изменять, запрашивать и удалять данные, а также определять структуру таблиц (например, с помощью команды «CREATE TABLE», как на Рис.3). Язык является декларативным, то есть указывает, что нужно получить, а не как это сделать. Операции с данными в SQL включают: SELECT – выборка данных: команда позволяет выбрать нужные данные из таблиц; Сортировка и фильтрация: ORDER BY – сортировка результатов, WHERE – фильтрация по условию, DISTINCT – исключение дубликатов; Агрегация: Функции COUNT(), SUM(), AVG() позволяют выполнять вычисления над данными; Добавление данных: INSERT INTO; Обновление данных: UPDATE; Удаление данных: DELETE; Управление структурой: CREATE TABLE – создание таблицы, DROP TABLE – удаление таблицы, ALTER TABLE – изменение структуры.

С развитием программных технологий возникла необходимость в создании универсальных интерфейсов, которые позволили бы приложениям взаимодействовать с разными СУБД без необходимости глубокого знания особенностей каждой из них. Для решения этой задачи были разработаны специализированные технологии: ODBC и JDBC.

- ODBC (Open Database Connectivity) - это API, разработанный Microsoft, который позволяет приложениям подключаться к различным базам данных (например, MySQL, PostgreSQL) через единый интерфейс. ODBC использует драйверы для взаимодействия с конкретной базой данных, обеспечивая независимость от СУБД.
- JDBC (Java Database Connectivity) - это API для языка Java, который позволяет Java-приложениям взаимодействовать с базами данных. Как и ODBC, JDBC предоставляет стандартный способ выполнения SQL-запросов и получения данных, но он специфичен для Java и работает через соответствующие драйверы.

2. Менеджер драйверов - библиотека, которая выбирает нужный драйвер в зависимости от типа БД.
3. Драйвер - модуль, преобразующий вызовы ODBC API в команды, понятные конкретной СУБД.
4. Источник данных (Data Source) - конкретная база данных, к которой происходит обращение.

Также существует несколько типов JDBC-драйверов:
1. JDBC-ODBC Bridge Driver - Использует мост между JDBC и ODBC. Java-приложение вызывает JDBC, который через ODBC-драйвер обращается к базе. Требует установленного ODBC-драйвера, легко реализуется, но зависит от ODBC;
2. Native-API Driver – Использует нативные библиотеки базы данных (например, DLL для Oracle). JDBC вызывает эти библиотеки для доступа к базе, быстрее 1 типа, но требует установки нативных библиотек, платформозависимый;
3. Network Protocol Driver (Middleware Driver) – Приложение через JDBC обращается к промежуточному серверу (middleware), который взаимодействует с базой. Middleware переводит вызовы в протокол базы, независим от базы, подходит для работы в сети, но нужен дополнительный сервер, сложнее в настройке;
3. Thon Driver - Напрямую взаимодействует с базой через сетевой протокол (например, TCP/IP). Написан полностью на Java, не требует дополнительных библиотек, высокая производительность, но специфичен для каждой базы.
