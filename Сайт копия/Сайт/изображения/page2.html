Список терминов:

1. База данных (БД) — совокупность структурно организованных данных, относящихся к некоторой предметной области, при этом вместе с данными хранится информация о характеристиках этих данных и об их взаимных связях.

2. Система управления базами данных (СУБД) — вычислительная система для создания и использования баз данных.

3. Реляционная база данных — база данных, основанная на реляционной модели данных, то есть основанная на организации данных в виде таблиц отношений.

4. API — программный интерфейс, который позволяет разным приложениям или сервисам обмениваться данными и функциями, например, позволяет приложению запросить данные с сервера или использовать функции другого сервиса, упрощая интеграцию.

5. Data Mapping (маппинг) — это процесс сопоставления данных между двумя разными системами или источниками. Он позволяет определить, какие данные из одного источника могут быть использованы в другом, а также как эти данные должны быть представлены и преобразованы для использования в другом контексте.

6. Фреймворк (с англ. framework — «каркас, структура») — заготовка, готовая модель в программировании для быстрой разработки, на основе которой можно дописать собственный код. Он задает структуру, определяет правила и предоставляет необходимый набор инструментов для создания проекта. В основном фреймворки используются в веб-разработке.

7. Mock — это тестовый объект, используемый в тестировании программного обеспечения для имитации поведения реальных компонентов системы, таких как базы данных или внешние API. Mock не содержит реальной логики и данных, но позволяет проверять взаимодействие тестируемого кода с зависимостями, гарантируя предсказуемые результаты и ускоряя выполнение тестов.
